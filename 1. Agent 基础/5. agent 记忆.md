# 1 记忆分类

- 在 AI Agent 的定义中，“记忆” 模块是支撑 Agent 实现长期连续交互和经验积累的核心，没有记忆的 Agent 每次处理请求都是 “失忆” 的，无法利用历史对话或过往任务的经验
- 根据存储时长和访问方式，通常将 Agent 记忆分为三类
  - 短期记忆
    - 时间跨度：单次会话（ 对话轮次内 ）
    - 记忆内容：当前用户的问题、Agent 的中间推理步骤、最近几次工具调用结果
    - 技术载体：内存中的消息列表、State 对象
  - 长期记忆
    - 时间跨度：跨会话、跨任务
    - 记忆内容：用户偏好、历史事实、过往成功/失败的策略、常见问题的解决方案
    - 技术载体：向量数据库、关系数据库、文件存储
  - 工作记忆
    - 时间跨度：当前任务执行期间
    - 记忆内容：当前任务的分解步骤、已完成的子目标、待验证的假设
    - 技术载体：State 中的临时字段，如 current_plan、completed_steps
- OpenAI 的技术定义中，“记忆” 主要对应短期记忆（ 保证连续对话 ）和长期记忆（ 实现知识沉淀 ），而工作记忆通常被归入 “规划” 模块

# 2. 短期记忆

## 2.1 基础介绍

- 短期记忆的目标是维护对话的上下文，让 Agent 能够理解 “刚才说了什么”
- 所有 LangChain/LangGraph 的 Agent 本质上都在维护一个 messages 列表，每个元素是一个 HumanMessage、AIMessage、ToolMessage
  ```python
  # 伪代码：Agent 内部的短期记忆
  state = {
    "messages": [
      HumanMessage(content="北京天气怎么样？"),
      AIMessage(content="", tool_calls=[{"name": "search", "args": {"city": "北京"}}]),
      ToolMessage(content="晴天 25°C", tool_call_id="..."),
      AIMessage(content="今天北京晴天，25°C。")
    ]
  }
  ```

## 2.2 压缩策略

- 当消息列表过长时，需要压缩短期记忆，常见策略如下
  - 按轮数截断：只保留最后 K 轮对话
  - 按 token 数截断：使用 ConversationTokenBufferMemory 保证总 token 不超过模型限制
  - 摘要替换：将早期对话交给 LLM 生成一段摘要，替换掉原始消息

## 2.3 具体实现

- LangGraph 使用 State 对象来维护短期记忆，State 对象在每次对话时（ 即调用 graph.invoke() 时 ）会重新创建（ 重置 ） State
- 但如果配置了 checkpointer 并且使用相同的 thread_id，LangGraph 会从检查点中恢复完整的 State 对象，包括 messages、user_id、current_step 等所有字段。第二次调用时传入的 initial_state 会与恢复的状态进行合并（ 通过各字段对应的 reducer ），因此其他未被传入的属性会保持上一次会话结束时的值
- 默认的 reducer 是覆盖，然而对 messages 我们预期是追加，因此通常需要通过 add_messages 去配置

  ```python
  from typing import Annotated, List
  from langgraph.graph.message import add_messages

  class AgentState(TypedDict):
    messages: Annotated[List, add_messages]   # add_messages 会追加而不是覆盖
    # 其他字段...
  ```

# 3. 长期记忆

## 3.1 基础介绍

- 长期记忆让 Agent 能够记住跨会话的信息，比如 “用户叫张三，喜欢简洁的回答”
- 技术架构：记忆存储 + 检索 + 注入
  ```
  用户输入 → 检索相关记忆 → 注入到上下文 → Agent 决策
                ↑
        向量数据库 / 关系库
  ```

## 3.2 记忆存储

- 基于向量数据库的记忆
  - 使用 embedding 模型将历史对话片段、用户偏好、知识条目向量化，存入向量库。每次用户输入时，将输入向量化并检索最相似的 K 条记忆，作为系统提示或附加消息

    ```python
    from langchain.memory import VectorStoreRetrieverMemory
    from langchain_community.vectorstores import Chroma
    from langchain_openai import OpenAIEmbeddings

    # 1. 创建向量存储
    embedding = OpenAIEmbeddings()
    vectorstore = Chroma(embedding_function=embedding)

    # 2. 将历史记忆存入
    memory = VectorStoreRetrieverMemory(retriever=vectorstore.as_retriever(k=3))
    memory.save_context({"input": "我叫小明"}, {"output": "好的，我会记住你叫小明。"})

    # 3. 在 Agent 中使用
    retrieved = memory.load_memory_variables({"input": "我是谁？"})
    # 输出: {"history": "用户叫小明"}
    ```

- 基于关系数据库的记忆
  - 适合存储结构化的用户属性或任务状态。例如用 SQLite 保存用户偏好
    ```sql
    CREATE TABLE user_memory (
        user_id TEXT,
        key TEXT,
        value TEXT,
        updated_at TIMESTAMP
    );
    ```
  - Agent 在每次交互前后读写该表
- 基于文件或知识库的记忆
  - 对于静态知识（ 如产品手册、公司政策 ），可以预先构建知识库，Agent 通过 RAG 检索

## 3.3 长期记忆的更新

- 显式更新：Agent 调用 update_memory 工具，主动写入记忆
- 隐式提取：由另一个专门负责 “记忆管理” 的 Agent 定期扫描对话历史，提取重要事实并存入长期记忆
- 时间衰减：给每条记忆设置权重，越久未被引用的记忆权重越低，定期清理

# 4. 记忆中间件

## 4.1 SummarizationMiddleware（ 短期记忆压缩 ）

- create_agent 提供了内置中间件 SummarizationMiddleware 来简化记忆管理，会自动触发摘要，将早期对话压缩成一段文本，保留关键信息

  ```python
  from langchain.agents.middleware import SummarizationMiddleware

  agent = create_agent(
    model="openai:gpt-4o",
    middleware=[SummarizationMiddleware(trigger_token_limit=8000)]
  )
  ```

## 4.2 自定义记忆中间件

- 可以自定义中间件，在 before_agent 钩子中从长期记忆加载相关内容，注入到 state
  ```python
  @before_agent
  def load_long_term_memory(state, runtime):
    user_id = state.get("user_id")
    memories = vectorstore.similarity_search(state["messages"][-1].content, k=3)
    memory_text = "\n".join([m.page_content for m in memories])
    # 注入到系统提示或消息列表
    state["messages"].insert(0, SystemMessage(content=f"相关记忆：{memory_text}"))
    return state
  ```

# 5. 设计要点

- 分层存储：短期（ 内存列表 ） + 长期（ 向量库/关系库 ） + 工作（ State 字段 ）
- 检索质量：使用好的 embedding 模型（ 如 `text-embedding-3-small` ），调优 top-k 和相似度阈值
- 更新机制：区分用户主动记忆和系统自动记忆，避免信息冲突
- 容量控制：长期记忆需要定期去重、压缩、删除低价值条目
- 隐私安全：敏感信息（ 如密码 ）不应存入长期记忆，或加密存储
