## 4. 内容审核

## 4.1 基础介绍

- 内容审核需覆盖输入侧（恶意提示、越狱尝试）和输出侧（有害内容、幻觉），且需在 Agent 的多步执行过程中持续进行，而非仅在聊天边界

## 4.2 核心防御措施

- Pre-LLM + Post-LLM Guardrails
  - Pre-LLM 防护在输入发送到模型前运行，处理 PII 检测、敏感数据阻止和提示注入检测；Post-LLM 防护在模型返回响应后运行，进行幻觉检测、毒性检测、工具和操作验证、输出格式合规检查
- 自校正循环
  - 当 Post-LLM guardrail 检测到问题时，不直接向用户返回失败，而是将标记的问题反馈给 LLM 进行针对性修正，Agent 重试后修正输出再次通过 guardrail
- 外部内容审核 API：集成专业内容审核服务对输入和输出进行毒性、偏见、仇恨言论等检测，结合人机协同机制实现人工监督/干预
- 平台级内容过滤器
  - Azure AI Agent Service 等在平台层面强制执行内容过滤器以确保安全和合规，需设计解决方案使 Agent 在检测到敏感意图时能正确调用自定义工具而非进入异常状态
