# 1. 什么是 Agent 安全

- Agent 安全关注的不是单纯 “模型会不会答错”，而是模型在具备工具、记忆和工作流能力之后，是否会被诱导去做 “错事”
- 与普通聊天机器人相比，Agent 不仅生成内容，还会调用外部能力
  - 读取网页、文档、数据库等外部数据
  - 调用工具执行查询、写入、发送、删除等动作
  - 把上下文写入记忆、日志、缓存或后续任务
- 因此，Agent 的风险会从 “内容问题” 升级为 “决策问题 + 执行问题 + 数据问题”

# 2. 为什么 Agent 安全更复杂

- Agent 安全之所以复杂，是因为它把模型、工具和数据流绑在了一起
- 一方面，LLM 无法像传统程序那样天然区分 “可信指令” 和 “不可信数据”
- 另一方面，Agent 的很多能力调用依赖模型的自然语言理解和自主决策，而不是完全由硬编码流程控制
- 同时，Agent 中的数据会在输入、推理、工具调用、记忆、日志和输出之间持续传播
- 这意味着一次错误判断，可能同时影响
  - Agent 听谁的
  - Agent 能做什么
  - Agent 接触和暴露了什么

# 3. Agent 安全主要关注什么

- Agent 安全主要聚焦三类问题
  - 提示注入防御：关注 Agent 是否会被恶意输入或外部内容劫持目标
  - 越权工具调用：关注 Agent 是否会调用不该调用的工具，或执行超出授权范围的动作
  - 敏感信息泄露：关注敏感数据是否会在上下文、工具链、记忆或日志中扩散
- 这三类问题分别对应三个层面
  - 控制面：Agent 最终听谁的
  - 执行面：Agent 实际能做什么
  - 数据面：Agent 接触和暴露了什么

# 4. 三者之间的关系

- 这三类问题不是彼此独立的，而是经常串成一条攻击链
- 常见路径是
  - 攻击者先通过提示注入影响 Agent 的目标和判断
  - 被影响的 Agent 再去调用本不该调用的工具
  - 工具读到或产出的敏感数据被进一步输出、记录或外发
- 所以可以把它们理解为
  - 提示注入 是入口
  - 越权工具调用 是执行放大器
  - 敏感信息泄露 是常见后果
